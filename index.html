Go Live 🚀
Deploy DayFlow to GitHub Pages — free, permanent URL
🌐
Your DayFlow URL will be:
https://YOUR-USERNAME.github.io/dayflow/
Free forever · Works on any device · No server needed · Updates instantly when you re-upload
1 Create a GitHub Account
If you don't have one yet, it's free and takes 2 minutes.
🐙 Create GitHub Account
2 Create a New Repository
1. Go to github.com → click "New repository" (the + button top right)
2. Name it exactly: dayflow
3. Set it to Public
4. Check "Add a README file"
5. Click "Create repository"
3 Upload Your DayFlow File
1. Inside your new repository, click "Add file" → "Upload files"
2. Drag and drop your DayFlow-2.0.html file
3. Rename it to index.html before uploading (right-click the file on your computer → rename)
4. Scroll down → click "Commit changes"
4 Enable GitHub Pages
1. In your repository, click Settings (top tab)
2. In the left sidebar, click "Pages"
3. Under "Source", select "Deploy from a branch"
4. Select branch: main · folder: / (root)
5. Click Save
6. Wait ~2 minutes, then your URL appears at the top of the Pages settings!
✅ You're Live!
Your DayFlow is now accessible from any device, anywhere — just bookmark the URL.

To update the app: simply upload a new index.html to the same repository and commit. GitHub Pages auto-publishes within 1-2 minutes.

Share with your other users: send them the URL. Each person creates their own profile on the login screen.
